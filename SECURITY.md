# Security Policy

## Supported Versions

Atualmente, apenas a versão mais recente do projeto recebe atualizações de segurança. Versões anteriores não são mantidas e podem conter vulnerabilidades conhecidas.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

> Recomendamos fortemente que todos os usuários mantenham seus sistemas atualizados com a versão mais recente disponível.

---

## Reporting a Vulnerability

Se você identificar uma vulnerabilidade de segurança no projeto **Smash or Pass**, siga as orientações abaixo para reportá-la de forma responsável:

### 📩 Como reportar

* Envie um e-mail para: **[arthurvininunes@gmail.com](mailto:arthurvininunes@gmail.com)**
* Ou abra uma issue privada (se disponível na plataforma)
* Evite divulgar publicamente a vulnerabilidade antes da correção

### 📝 O que incluir no relatório

Para facilitar a análise, inclua:

* Descrição detalhada da vulnerabilidade
* Passos para reprodução
* Possível impacto
* Sugestões de correção (se houver)
* Prints ou logs relevantes (opcional)

### ⏱️ Tempo de resposta

* Confirmação de recebimento: até **48 horas**
* Análise inicial: até **5 dias úteis**
* Atualizações contínuas conforme o progresso da correção

### 🔒 Processo de tratamento

* A vulnerabilidade será analisada e classificada por severidade
* Caso confirmada, será corrigida o mais rápido possível
* Um patch será publicado junto com a atualização de versão
* O responsável pelo reporte poderá ser creditado (opcional)

### ❌ Vulnerabilidades não aceitas

Relatórios podem ser recusados caso:

* Não sejam reproduzíveis
* Sejam relacionados a versões não suportadas
* Sejam considerados de baixo impacto ou comportamento esperado

---

## Boas práticas de segurança

* Nunca exponha dados sensíveis no código
* Utilize variáveis de ambiente para configurações críticas
* Valide entradas de usuários
* Mantenha dependências atualizadas
